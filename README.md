# Workers 探针方案优势概述

基于 Cloudflare Workers 的探针解决方案，兼具高可用性、低成本和模块化可扩展性。下文汇总了该方案的核心卖点，便于快速向团队或潜在用户介绍其价值。

## 1. 完全 Serverless，降低运维成本
- 运行在 Cloudflare Workers 上，不占用自有 VPS 资源。
- 即便所有节点下线，控制面板仍保持可用，能够查看最新心跳记录。
- 无需自建后端或数据库，依靠 Cloudflare 免费/低价档位即可运行。

## 2. 单探针覆盖 VPS 与站点监控
- 支持 VPS 级监控：心跳、CPU、内存、磁盘、网络等指标。
- 支持网站/接口监控：HTTP 状态码、响应时间、宕机检测等。
- 一个后台统一展示所有资源的健康状态，无需拆分多个系统。

## 3. 双重体系保障安全
- 管理后台采用账号 + 密码 + JWT，具备登录失败限制、账号锁定和审计能力。
- 探针 Agent 使用 API_KEY 上报，一机一 Key，可细粒度控制权限、限流与吊销。
- 仅需公网 HTTPS 通信，无需暴露额外端口。

## 4. 部署体验友好
- 后台为每个 VPS 监控对象自动生成 API_KEY。
- 前端提供“一键安装脚本”，一次执行即可完成探针安装与定时任务配置。
- 适合普通运维或小白用户，扩容新节点只需几次点击与复制命令。

## 5. 灵活高效的告警机制
- 支持按策略触发 Telegram 告警：如 VPS 掉线、站点宕机、资源超阈值等。
- 可配置冷却时间与重复告警策略，避免消息轰炸。
- 后续可扩展更多告警渠道和指标，满足多样化需求。

## 6. 直观友好的 UI
- 类卡片式面板展示每台 VPS/站点的在线状态与核心指标。
- 提供 CPU、内存、磁盘进度条和网络速率等关键信息。
- 支持快速浏览整体状况，并逐步扩展历史趋势与详细视图。

## 7. 架构清晰，模块解耦
- 模块包括 Auth、Agent、Site Monitor、Alert、API_KEY、UI 等，职责明确。
- 扩展新特性（多种告警渠道、更多指标、第三方集成）无需大规模重构。
- 干净的逻辑和数据模型便于后续开源、商用或二次开发。

## 8. 运维与可运营性兼顾
- 提供 healthz/readyz 等健康检查接口，支持登录审计和操作留痕。
- 保留告警事件记录，跟踪具体节点的故障时间线。
- 规划历史数据保留策略，避免 D1 体量膨胀，同时支持 user_id 维度的多租户隔离。

## 总结
依托 Workers、D1 与轻量级 Agent，整个探针系统无需复杂后端即可实现高可用、好用且可扩展的多功能监控平台，兼顾部署便捷、运维友好和未来迭代空间。
